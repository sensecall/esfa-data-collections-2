Validation errors - Bank details

Modifies what they enter in the account number field.


Date of release
1 June 2017
Feature
Payment Scheduler
Related stories
BPS-181

Changes made

Prototype designed to test the validation messaging on the bank details. When submitting the form for the first time, if there is 8 or more characters in the account number box, a function will switch out the last character that has been entered, effectively creating a typo. This enables us to test the validation messages in a more realistic environment.

Research notes

As soon as the error message was presented agents presumed that they had done something wrong, when following the message down to the relevant fields the 'enter these details again' was taken literally by some agents so rather than checking the information it was deleted out and fully rekeyed. The other outcome was that agents checked the information on the claim form against the information they had entered and rectified the error. The initial message stating that the details entered were not a valid combination prompted agents to check the information provided by the claimant, to request correct information if necessary and if the problem had not been solved at this point to contact the bank liaison team.
