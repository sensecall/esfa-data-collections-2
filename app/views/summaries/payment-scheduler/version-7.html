Validation errors

Have all payments upto and including [Month] been made?

Changes made

Changed the content asking the agent if they have made all of the clerical payments to date. Added in the overseas country selector so that it could be tested with IPC.

Research notes

Agents were aware that payments were being sent to the Citi sort code and account number and this caused confusion when asking agents to then type in the claimant bank details. This may have been because of the country selector as there was some hesitation here. When viewing a payment schedule, agents found that they needed to be able to see the claimant bank details to help with enquiries as a lot of enquiries are related to payment dates and payment details. They did not see this task as unnecessary duplication.

There was no hesitation when asking if all payments were up to date, however it became apparent that agents were not reading this information as only providing one option led them to continue regardless. When asked what action they would take if the payments weren't up to date, it was expected that BSP Manager would pay any arrears/ missed payments as agents thought they could select a 'no' option to tell the system what needed to be paid. They were not aware that any missed payments would have to be paid manually.
